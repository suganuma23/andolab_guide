研究室PC

  • 研究室のPCは Ubuntu (Linux) がメインOS
  • ユーザー名・パスワードは先輩または先生から直接聞いてください（本サイトには掲載しません）
  • ハンガーは右から3番目のロッカーにあります

電気について

研究室の電気スイッチは古いタイプのボタン式です。場所がわからなければ先輩に聞いてください。

印刷機の設定（Linux）

研究室の印刷機を Linux PC から使うための手順です。

  1. linux-lipslx-drv-v610-jp-00 というドライバーをネットでダウンロードする
  2. ダウンロードしたフォルダに移動してインストールスクリプトを実行する： 
    cd ダウンロードしたフォルダ
sudo ./install.sh
  3. 設定ウィンドウが開くので、プリンター名を追加する
  4. プリンターのIPアドレスは先輩に確認してください

トラブルシューティング

印刷できない場合は、プリンターの電源が入っているか・ネットワークに接続されているかを確認してください。

SSH / VPN

SSH鍵の設定

  1. 鍵ペアを生成する（鍵の名前に接続先サーバ名を入れておくと管理しやすい）： 
    ssh-keygen -t ed25519
  2. 公開鍵・秘密鍵は ~/.ssh/ に置く
  3. 公開鍵をサーバーに送信し、authorized_keys に追加する
  4. ~/.ssh/config にサーバーの接続設定を書く： 
    Host サーバ名
    HostName 接続先アドレス
    User ユーザー名
    IdentitiesOnly yes
    IdentityFile ~/.ssh/秘密鍵ファイル名
  5. これで ssh サーバ名 だけで接続できるようになる

VPN

  • 学外から大学ネットワークにアクセスする場合はVPNが必要
  • VPNの設定方法は大学の情報基盤センターのページを参照

Tailscale（VPNツール）

研究室では Tailscale を使ってリモートSSH接続することもあります。

  1. 各PCで Tailscale をインストールし、同じアカウントでログインする
  2. Tailscale を起動する
    • Ubuntu: sudo tailscale up
    • Windows: アプリを開いて起動
  3. SSHしたいサーバのVPN上のIPアドレスをコピーして接続する

VSCode（エディタ）

研究室での開発には主に Visual Studio Code を使います。

基本知識

  • VSCode はWindows上のソフトウェア
  • WSL上の Ubuntu でも開けるのは、Ubuntu上にサーバーを置いて接続しているため
  • Remote-SSH 拡張機能を使えば、研究室サーバにSSH接続してリモート開発できる

便利なショートカットキー

ショートカット動作

Ctrl + Alt + SRemote-SSH接続
Ctrl + Alt + WWSL接続
Ctrl + Alt + Q接続を切る

設定ファイル

  • settings.json で拡張機能やエディタの設定を記述できる
  • > Open Remote Setting でリモート用設定を開ける
  • Open Default Setting で全設定（約8000行）が開く
  • Open User Setting でGUIの設定画面が開く

tmux（ターミナル多重化ツール）

tmux は、1つのターミナル（特にSSH接続）上で複数の仮想端末を管理できるツールです。研究で長時間かかる処理を実行するときに必須です。

tmuxでできること

  • SSH接続が切れてもプログラムを実行し続けられる
  • 複数の作業画面（タブのようなもの）を切り替えられる
  • 画面を分割して同時に複数の作業ができる
  • 作業状態を保持したまま離脱（detach）し、後で再接続できる

重要な注意点

tmux は「SSH切断に強い」だけです。ローカルPCをスリープすると処理は止まります。長時間処理は必ずスリープしないマシン（サーバ）上で実行してください。

3階層構造

概念説明
セッション作業単位の一番大きな枠。SSH切断しても残る。最も重要な概念
ウィンドウタブのようなもの。用途ごとに分ける（run, log, monitor など）
ペインウィンドウ内を上下・左右に分割した画面

長時間処理の実行手順

  1. SSH先のサーバ上で tmux セッションを作成する
  2. セッション内でプログラムを実行する
  3. detach して SSH を切断する（exit ではなく detach が正解）
  4. 後で再接続して結果を確認する

操作ごとの結果

操作結果
detachセッションは残る（安全）
exitそのウィンドウの bash が終了。最後のウィンドウならセッションも終了
SSH 切断tmux があれば問題なし
ローカルPC スリープ処理は必ず止まる

Git / GitHub

研究室のソースコードは Git で管理し、GitHub で共有します。

GitHubへのSSH接続

  1. SSH鍵を生成する（GitHub用だとわかる名前にする）
  2. 公開鍵を GitHub にアップロードする
  3. ~/.ssh/config に設定を記述する
  4. 接続テスト: ssh -T git@github.com

clone するときは SSH の URL を使う。HTTPS にしてしまった場合：

git remote set-url origin git@github.com:ユーザー名/リポジトリ.git

よく使う Git コマンド

コマンド説明

git pullリモートから取得・統合

設定ファイル

  • ~/.gitconfig にエイリアスやアカウントの設定が書いてある

設定ファイルの場所

  • ホームディレクトリ (~) に各種設定ファイルがある
  • 主な設定ファイル：.bashrc（シェル）, .vimrc（Vim）, .gitconfig（Git）, .tmux.conf（tmux）
  • . で始まるファイルは隠しファイル。ls -a で表示できる
  • ~/.ssh/ に公開鍵・秘密鍵を置く。~/.ssh/config にはSSHするサーバーの設定を書く
