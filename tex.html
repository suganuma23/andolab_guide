テンプレートの入手

報告書・卒論はすべてLaTeXで作成します。

テンプレートの入手先

  • 報告書の例: report_example.tex（右クリックで保存）
  • ゼミ・卒論のフォーマット: Teamsの共有ファイル

基本テンプレート

以下は報告書の基本的な構造です。実際に使うときは上記のテンプレートに従ってください。

\documentclass[a4paper,11pt]{jsarticle}
\usepackage[dvipdfmx]{graphicx}
\usepackage{amsmath,amssymb}

\title{週次報告書}
\author{氏名}
\date{\today}

\begin{document}
\maketitle

\section{今週の進捗}
ここに内容を書く.

\section{来週の予定}
ここに予定を書く.

\end{document}

安藤研特有のTeXルール

安藤研では以下のルールを守ってTeXを書きましょう。これを守らないと添削で指摘されます。

1. キャプションの後にはピリオドをつける

図や表のキャプションの末尾には必ず .（ピリオド）をつけます。

% OK
\caption{実験結果の比較.}

% NG
\caption{実験結果の比較}

2. 句読点は「,」と「.」を使う

日本語の文章でも、「、」と「。」は使わず、「,」と「.」（半角カンマとピリオド）を使います。これは学術論文の慣習です。

% OK
本研究では,グラフ理論を用いた手法を提案する.

% NG
本研究では、グラフ理論を用いた手法を提案する。

3. 半角文字と全角文字の間にはスペースを入れる

半角英数字と全角文字（日本語）が続くときは、半角スペースを1つ空けると読みやすくなります。

% OK
本研究では Graph Theory を用いる.

% NG
本研究ではGraph Theoryを用いる.

まとめ

  • キャプション末尾 → . をつける
  • 句読点 → , と . を使う（「、」「。」は不可）
  • 半角と全角の境界 → 半角スペースを入れる

よく使うパッケージ

パッケージ用途使い方の例
graphicx 図の挿入 \usepackage[dvipdfmx]{graphicx}
amsmath 高度な数式環境 \usepackage{amsmath}
amssymb 数学記号の拡張 \usepackage{amssymb}
amsthm 定理環境（定理・補題・証明など） \usepackage{amsthm}
hyperref ハイパーリンク・PDF目次の生成 \usepackage[dvipdfmx]{hyperref}
enumerate 番号付きリストのカスタマイズ \usepackage{enumerate}
tikz グラフ・図形の描画 \usepackage{tikz}
algorithm2e 擬似コード（アルゴリズム）の記述 \usepackage[ruled]{algorithm2e}
booktabs 美しい表の作成 \usepackage{booktabs}

パッケージは \documentclass の後、\begin{document} の前に記述します。共有ファイルのテンプレートにはよく使うものが最初から入っています。

図の挿入方法

図を挿入するには figure 環境を使います。以下が基本的な書き方です。

\begin{figure}[htbp]
  \centering
  \includegraphics[width=0.8\linewidth]{fig/result.png}
  \caption{実験結果のグラフ.}
  \label{fig:result}
\end{figure}

ポイント

  • [htbp] は図の配置オプション（here, top, bottom, page の順に試行）
  • \centering で図を中央揃えにする
  • width=0.8\linewidth で幅を本文の80%に設定（適宜調整）
  • \caption の末尾には 必ずピリオド をつける（安藤研ルール）
  • \label をつけておくと、本文中で \ref{fig:result} や 図\ref{fig:result} のように参照できる

図を横に並べる場合

\begin{figure}[htbp]
  \centering
  \begin{minipage}{0.48\linewidth}
    \centering
    \includegraphics[width=\linewidth]{fig/before.png}
    \caption{処理前の画像.}
    \label{fig:before}
  \end{minipage}
  \hfill
  \begin{minipage}{0.48\linewidth}
    \centering
    \includegraphics[width=\linewidth]{fig/after.png}
    \caption{処理後の画像.}
    \label{fig:after}
  \end{minipage}
\end{figure}

参考文献（BibTeX）の書き方

参考文献は BibTeX を使って管理するのが便利です。.bib ファイルに文献情報を書き、本文から \cite{} で引用します。

.bib ファイルの書き方

references.bib などのファイル名で保存します。

% 論文（Article）
@article{ando2020spanning,
  author  = {Kiyoshi Ando and Atsushi Kaneko},
  title   = {Spanning trees with bounded number of leaves},
  journal = {Discrete Mathematics},
  volume  = {343},
  number  = {6},
  pages   = {111842},
  year    = {2020}
}

% 書籍（Book）
@book{diestel2017graph,
  author    = {Reinhard Diestel},
  title     = {Graph Theory},
  edition   = {5th},
  publisher = {Springer},
  year      = {2017}
}

% 学会発表（InProceedings）
@inproceedings{suzuki2023algorithm,
  author    = {Taro Suzuki and Kiyoshi Ando},
  title     = {An efficient algorithm for tree enumeration},
  booktitle = {Proceedings of the 10th Conference on Graph Theory},
  pages     = {123--130},
  year      = {2023}
}

本文での引用と参考文献リストの出力

% 本文中での引用
Ando and Kaneko \cite{ando2020spanning} は全域木について...

% 文末に参考文献リストを出力
\bibliographystyle{plain}
\bibliography{references}

BibTeX のコンパイル手順

  1. platex main.tex（1回目）
  2. pbibtex main（BibTeX処理）
  3. platex main.tex（2回目: 参照の解決）
  4. platex main.tex（3回目: 参照の確定）
  5. dvipdfmx main.dvi（PDF生成）

MathSciNet から BibTeX データを直接コピーできるので、論文検索時に活用しましょう。

数式の書き方

インライン数式

文中に数式を入れるには $...$ で囲みます。

グラフ $G = (V, E)$ の頂点数を $n = |V|$ とする.

ディスプレイ数式（番号付き）

\begin{equation}
  \sum_{i=1}^{n} d(v_i) = 2|E|
  \label{eq:handshake}
\end{equation}

ディスプレイ数式（番号なし）

\[
  \binom{n}{k} = \frac{n!}{k!(n-k)!}
\]

複数行の数式（align 環境）

\begin{align}
  f(x) &= (x+1)^2 \\
       &= x^2 + 2x + 1
  \label{eq:expansion}
\end{align}

場合分け（cases 環境）

\[
  f(x) = \begin{cases}
    1  & \text{if } x > 0 \\
    0  & \text{if } x = 0 \\
    -1 & \text{if } x < 0
  \end{cases}
\]

定理環境の例

% プリアンブルで定義
\newtheorem{theorem}{定理}
\newtheorem{lemma}[theorem]{補題}

% 本文で使用
\begin{theorem}
  任意の木 $T$ について, $|E(T)| = |V(T)| - 1$ が成り立つ.
  \label{thm:tree-edges}
\end{theorem}

\begin{proof}
  頂点数 $n$ に関する帰納法で示す.
  ...
\end{proof}

AI を活用した TeX 執筆 Tips

CLAUDE.md にルールを書いておけば AI が勝手に修正してくれる

プロジェクトのルートに CLAUDE.md ファイルを作成し、安藤研の TeX ルール（句読点の変換、キャプションのピリオド、スペースの挿入など）を記述しておくと、Claude などの AI ツールがそのルールに従って自動的に修正・提案してくれます。

例えば、CLAUDE.md に以下のように書いておきます。

# LaTeX 執筆ルール（安藤研）

- 句読点は「、」「。」ではなく「,」「.」を使うこと
- キャプションの末尾には必ず「.」をつけること
- 半角文字と全角文字の間には半角スペースを入れること
- 参考文献は BibTeX で管理すること
- 図の label は fig: で始めること（例: fig:result）
- 表の label は tab: で始めること（例: tab:comparison）
- 式の label は eq: で始めること（例: eq:handshake）

これにより、TeX ファイルを AI に渡したときにルール違反を自動検出・修正してもらえます。手動でチェックする手間が大幅に減るので、ぜひ活用してください。
