TeXとは

TeX（テフ）は、数式を含む文書を美しく組版するためのシステムです。安藤研では報告書・卒論・アブストラクト・スライドの全てをTeXで作成します。

なぜTeXを使うのか

  • 数式が美しい — Word等とは比較にならない品質で数式が組版される
  • バージョン管理と相性が良い — テキストファイルなのでGitで管理できる
  • 学術論文の標準 — 数学・情報系の論文はほぼ全てTeXで書かれている

TeXの環境構築

TeX Liveのインストール

TeXのディストリビューションとしてTeX Liveを使います。

Ubuntu / WSL:

sudo apt install texlive-full

texlive-full は容量が大きい（数GB）ですが、パッケージ不足で悩まなくて済むのでおすすめです。

インストール確認

platex --version
dvipdfmx --version

安藤研では platex（日本語対応）→ dvipdfmx（PDF変換）の流れでコンパイルします。

コンパイル方法

基本: ptex2pdf（おすすめ）

ptex2pdf を使えば、platex → dvipdfmx を1コマンドでPDFまで生成できます。

ptex2pdf -l report.tex

-l オプションは platex を使う指定です。これだけで report.pdf が生成されます。

相互参照（\ref や \cite）がある場合は2回実行すると正しく解決されます。

ptex2pdf -l report.tex
ptex2pdf -l report.tex

手動でステップを分ける方法

ptex2pdf が内部でやっていることを手動で行う方法です。エラーの原因を特定したいときに便利です。

# 1. platex でコンパイル（.dvi を生成）
platex report.tex

# 2. DVI を PDF に変換
dvipdfmx report.dvi

その他のコンパイル方法

方法コマンド特徴
ptex2pdf ptex2pdf -l file.tex platex + dvipdfmx を一括実行。安藤研の標準
platex + dvipdfmx platex file.tex
dvipdfmx file.dvi		 手動2ステップ。デバッグ時に有用
latexmk latexmk -pdfdvi file.tex 依存関係を自動解析し必要な回数だけコンパイル。参照も自動解決
lualatex lualatex file.tex 直接PDF生成。Unicode対応が強い。安藤研では使わない
pdflatex pdflatex file.tex 直接PDF生成。英語論文で主流。日本語非対応

latexmk の設定（便利）

~/.latexmkrc に以下を書いておくと、latexmk file.tex だけでplatex + dvipdfmx が走ります。

# ~/.latexmkrc
$latex = 'platex -interaction=nonstopmode -halt-on-error %O %S';
$dvipdf = 'dvipdfmx %O -o %D %S';
$pdf_mode = 3;  # platex + dvipdfmx

さらに -pvc オプションでファイル変更を監視し、保存するたびに自動コンパイルもできます:

latexmk -pvc file.tex

% 数式の例
\begin{equation}
  f(x) = \sum_{i=1}^{n} w_i |x_i - y_i|^2
\end{equation}

% 図の挿入

参考文献の書き方

安藤研では thebibliography 環境と \bibitem を使って参考文献を管理します。

基本形

\begin{thebibliography}{00}

\bibitem{ラベル名}
著者名: タイトル. \textit{雑誌名} \textbf{巻} (年) ページ.

\end{thebibliography}

{00} は参考文献の数が2桁であることを示します（文献数が多いときは {99} にする）。

論文の場合

\bibitem{pardalos1999}
P.M. Pardalos and G. Xue: Algorithms for a class of isotonic
regression problems. \textit{Algorithmica} \textbf{23} (1999) 211--222.

形式: 著者: タイトル. \textit{雑誌名} \textbf{巻} (年) ページ.

書籍の場合

\bibitem{semple2003}
C. Semple and M. Steel: \textit{Phylogenetics}
(Oxford University Press, 2003).

形式: 著者: \textit{書名} (出版社, 年).

修士論文・卒業論文の場合

\bibitem{ishida25}
石田京太郎: 重み付き$\ell_1$最良近似超距離木問題に対する
部分木交換操作に基づく局所探索アルゴリズム.
修士論文. 静岡大学大学院総合科学技術研究科, 2025年2月.

形式: 著者: タイトル. 修士論文. 大学名, 年月.

本文中での引用

Pardalosら\cite{pardalos1999}は、半順序上の単調回帰問題に対して...

\cite{ラベル名} と書くと、自動的に [1] のように番号が振られます。

ラベル名は自由に決められますが、著者名+年（例: pardalos1999）や著者イニシャル+年（例: AC11）が一般的です。

安藤研TeXフォーマットメモ

安藤研のTeXには独自のルールやスタイルファイルがあります。文書の種類ごとにまとめます。

ルールファイルのダウンロード

以下のmdファイルをダウンロードして、TeXリポジトリのCLAUDE.mdに記載しておくと、AIエージェント（Claude Code等）がTeXの修正時にこのルールに従ってくれます。

andolab_tex_rules.md

共通ルール

句読点・記号

項目ルール例
句読点「、」「。」は使わない。「,」「.」に統一系統樹を推定することは, 重要な課題である.
読点の後半角スペースを入れるここで, $n = |X|$ である.
文末ピリオドの前スペースなし...で解くことができる.
キャプション末尾必ずピリオド「.」をつける\caption{グラフ $G$ の構造.}

スペーシング

項目ルール例
半角と半角の間半角スペースを入れる$k$SS操作 (Subtree Swap Operation)
日本語と数式の間スペースなしでも可（TeXが自動調整）問題は$\ell_2$単調回帰問題と呼ばれる.
英単語と日本語の間スペースなしでも可PAVアルゴリズムの一般化
引用の前~（改行禁止スペース）を使うAilon et al.~\cite{AC11}は
括弧の外側半角スペースを入れるSS操作 (Subtree Swap Operation) と呼ぶ.

数式の書き方

項目ルール例
ローマン体の関数名\mathrm{} を使う$\mathrm{O}(n \log n)$
太字ベクトル\bm{} や \mathbb{}$\mathbb{R}_+$
ノルム\|\cdot\|$\|D_{(T,h)} - M\|_2$
条件付き最適化\mathrm{s.t.}\mathrm{s.t.} \quad h \in \mathbb{R}^V
テキスト中の条件\text{} を使う\text{if } p < \infty

図・表・アルゴリズム

項目ルール
画像形式EPS形式を使う（tgif / gnuplot で生成）
画像挿入\usepackage[dvipdfmx]{graphicx} を使う
図のフォルダtgif図は tgifs/、gnuplot図は gnuplots/ に置く
図の位置指定[H]（here パッケージ）または [htbp]
サブキャプションsubcaption パッケージ + minipage で並べる
表の罫線booktabs パッケージの \toprule, \midrule, \bottomrule を使う

参照・引用

項目ルール例
図の参照図\ref{fig:xxx}（スペースなし）図\ref{fig:ctree}に示す.
式の参照式\eqref{eq:xxx} を使う式\eqref{eq:butt}は...
表の参照表\ref{tab:xxx}表\ref{tab:time}にまとめた.
アルゴリズム参照アルゴリズム\ref{alg:xxx}アルゴリズム\ref{alg:kss}は...
文献引用\cite{xxx}、複数著者は~で繋ぐ石田\cite{ishida25}は...
ラベル命名規則図: fig:、式: eq:、表: tab:、アルゴリズム: alg:

文体・構成

項目ルール
文体「である・だ」調（敬体は使わない）
段落区切り\par で明示的に段落を区切ることが多い
定理環境定理・補題・命題・定義を \newtheorem で定義して使う
証明自分で \proof 〜 \endproof を定義するか、amsthmを使う
太字で用語定義初出の用語は \textbf{} で太字にする
参考文献thebibliography + \bibitem を使う（BibTeX不使用）

参考文献の書式

thebibliography 環境 + \bibitem で書く（BibTeXは使わない）。書式は文献の種類で異なる。

論文の場合:

著者名: タイトル. {\it 雑誌名} {\bf 巻} (年) ページ.

実例:

\bibitem{AC11}
N. Ailon and M. Charikar: Fitting tree metrics: hierarchical
clustering and phylogeny.
{\it SIAM Journal on Computing} {\bf 40} (2011) 1275--1291.

書籍の場合:

著者名: \textit{書名} (出版社, 年).

実例:

\bibitem{semple2033}
C. Semple and M. Steel: \textit{Phylogenetics}
(Oxford University Press, 2003).

修士論文・卒業論文の場合:

著者名: タイトル. 修士論文. 大学名, 年月.

実例:

\bibitem{ishida25}
石田京太郎: 重み付き$\ell_1$最良近似超距離木問題に対する
部分木交換操作に基づく局所探索アルゴリズム.
修士論文. 静岡大学大学院総合科学技術研究科, 2025年2月.
項目ルール
雑誌名イタリック体: {\it 雑誌名} または \textit{雑誌名}
巻番号太字: {\bf 巻} または \textbf{巻}
ページ範囲enダッシュ: 211--222（ハイフン2つ）
著者の区切りand で繋ぐ（3人以上でも全員記載）
著者とタイトルの間コロン + スペース: 著者: タイトル.
並び順アルファベット順（著者の姓で）
ラベル名著者イニシャル+年（例: AC11）または著者名+年（例: pardalos1999）

報告書

進捗報告のたびに作成するTeXファイルです。

ドキュメントクラスとパッケージ

\documentclass[a4paper,dvipdfmx]{jsarticle}
\usepackage{amsmath,amssymb}
\usepackage{graphicx}
\usepackage[lined,linesnumbered,boxed]{algorithm2e}
\usepackage{here}
\usepackage{comment}
\usepackage{booktabs}
\usepackage{diagbox}

\SetAlgorithmName{アルゴリズム}{アルゴリズム}{アルゴリズムのリスト}
\SetKwInput{KwIn}{入力}
\SetKwInput{KwOut}{出力}
  • クラス: jsarticle（日本語記事クラス）
  • スタイルファイル: 不要
  • 構造: \section で節を分ける
  • 報告書例: report_example.zip

アブストラクト（ゼミ・卒研発表会）

ゼミアブストラクトと卒研アブストラクトは同じフォーマットです。

ドキュメントクラスとスタイルファイル

\documentclass[9pt,a4paper,twocolumn]{jarticle}
\usepackage{chukan}
\author{菅沼 良規}
\gakuseki{5021-6027}
\title{タイトル}
\lab{安藤研究室}
\date{}
  • クラス: jarticle（9pt, A4, 2段組）
  • スタイルファイル: chukan.sty（専用のスタイルファイル。TeXファイルと同じディレクトリに置く）
  • \gakuseki{} で学籍番号、\lab{} で研究室名を指定
  • A4 1ページに収める
  • ゼミアブストのタイトルには \footnotemark[1] で英語論文の情報を脚注に入れる

chukan.styのフォーマットは Teamsの共有ファイルにあります。先輩の実例は andolab_report_b4.zip にも含まれています。

卒業論文

卒論は報告書やアブストラクトとは異なるクラス・スタイルファイルを使います。

ドキュメントクラスとスタイルファイル

\documentclass[11pt]{jreport}
\usepackage{systhesisu}
\usepackage{amsmath}
\usepackage[lined,linesnumbered,boxed]{algorithm2e}
\usepackage{graphicx}
\usepackage[dvipdfmx]{color}
\usepackage[margin=30mm]{geometry}

\SetAlgorithmName{アルゴリズム}{アルゴリズム}{アルゴリズムのリスト}
\SetKwInput{KwIn}{入力}
\SetKwInput{KwOut}{出力}
\renewcommand{\bibname}{参考文献}

\title{タイトル}
\author{菅沼  良規}
\advisor{安藤  和敏}{教授}
\reiwayear{7}
\date{2026 年 2 月}
  • クラス: jreport（11pt、章立て構造）
  • スタイルファイル: systhesisu.sty（卒論専用。TeXファイルと同じディレクトリに置く）
  • \advisor{} で指導教員名、\reiwayear{} で令和の年を指定
  • 構造: \chapter で章を分ける（報告書の \section とは異なる）
  • 目次: \tableofcontents で自動生成
  • 参考文献: \renewcommand{\bibname}{参考文献} で日本語化
  • 参考文献を目次に追加: \addcontentsline{toc}{chapter}{参考文献}

systhesisu.styのフォーマットは Teamsの共有ファイルにあります。先輩の実例は andolab_report_b4.zip にも含まれています。

ゼミ発表資料（本体）

英語論文を日本語でまとめた発表資料です。

ドキュメントクラス

\documentclass[11pt]{jarticle}
\usepackage{enumerate}
\usepackage{amssymb}
\usepackage{amsmath}
\usepackage[lined,linesnumbered,boxed]{algorithm2e}
\usepackage[dvipdfmx]{graphicx}
  • クラス: jarticle（11pt）
  • スタイルファイル: 不要
  • 構造: \section で節を分ける
  • 余白は手動で設定: 
    \setlength{\topmargin}{-0.7cm}
\setlength{\oddsidemargin}{0cm}
\setlength{\textwidth}{15.9cm}
\setlength{\textheight}{23.4cm}
  • 定理環境を自分で定義する: 
    \newtheorem{theorem}{定理}[section]
\newtheorem{lemma}[theorem]{補題}
\newtheorem{proposition}[theorem]{命題}
\newtheorem{defe}{定義}[section]
