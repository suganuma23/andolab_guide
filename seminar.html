ゼミの概要

安藤研のゼミは、4年生が英語論文を読み解き、それを和訳・発表するスタイルです。5月〜7月にかけて毎週行われます。

  1. 4年生にはそれぞれ英語論文が1本渡される
  2. 毎週、その論文の決められた範囲を和訳し、TeXの文章にまとめる
  3. 輪講室で発表する（先生・先輩と4年生1人の前で）
  4. 先生からフィードバックを受ける

TeXフォーマットについて

ゼミ発表用のTeXフォーマットは andolab_report_b4 リポジトリの「ゼミ」フォルダにあります。まずこのリポジトリをクローンして使いましょう。

git clone git@github.com:<組織名>/andolab_report_b4.git

ゼミのスケジュール

時期内容
5月中旬ゼミの日程打ち合わせ（前年実績: 5月13日）
5月〜7月週1回のゼミ発表（英語論文の和訳・発表）
7月中旬アブストラクト提出期限（前年実績: 7月16日）
7月中旬〜下旬発表練習（前年実績: 7月14日〜18日）
7月下旬セミナー発表会（前年実績: 7月19日 終日）

毎週の準備の流れ

  1. 担当範囲を確認する — 先生から指定された論文の範囲（例: Section 2.1〜2.3）
  2. 論文を読む — まず英語のまま内容を理解する。定義・定理・証明の流れを把握する
  3. 和訳をTeXで書く — andolab_report_b4のフォーマットに従ってTeXファイルを作成する。数式はそのまま写すのではなく、意味を理解した上で記述する
  4. 発表資料として印刷する — ゼミ当日に配布できるように印刷しておく
  5. 輪講室で発表する — 和訳した内容を口頭で説明する

重要: 単なる機械的な和訳ではなく、内容を自分の言葉で説明できるレベルまで理解することが求められます。「ここはどういう意味？」と聞かれて答えられないと怒られます。

アブストラクト・セミナー発表会

アブストラクトの作成

7月頃から、ゼミで読んだ論文の内容をアブストラクト（要約）にまとめます。

  • フォーマットは andolab_report_b4 リポジトリにある
  • 論文の背景・目的・主要な結果・結論を簡潔にまとめる
  • 提出期限を厳守すること（前年実績: 7月16日）

スライドの作成

アブストラクトが完成したら、セミナー発表会に向けてスライドを作成します。

  • スライドのサイズは通常サイズ（ワイドではない）
  • 内容は論文の要約＋自分の理解を反映させる
  • ゼミでの先生のフィードバックを反映させること

発表練習

セミナー発表前の一週間は発表練習を行います。

  • 前年実績: 7月14日〜18日（18日は全員の前で通し練習）
  • 先生からフィードバックを受けて、スライドを修正する
  • 時間配分も含めて練習すること

発表のコツ

スライド作成のポイント

  • ビジュアルでなるべく表現する — 文字は限界まで減らす。図や具体例を多用する
  • 統一する — フォント、サイズ、図の位置、図のサイズなど。無意味に変更しない
  • 矢印は変化と因果関係に使う — 装飾的な矢印は使わない
  • スライドのサイズは通常サイズ — ワイドスライドは使わない

話し方のポイント

  • 時間がなければ、定義・定理と例を同時に説明する（定義だけ先に並べない）
  • 聞いている人への動機づけを意識する — 「もっと知りたい」と思わせるように話す
  • 単に理解した内容を綺麗にまとめるだけでは不十分。聞き手の興味を引く工夫が求められる
  • 「感謝の気持ち」を持って発表する — めんどくさいなどの態度は絶対NG

発表時の持ち物

  • アブストラクト（印刷したもの）
  • 添削資料（先生からのフィードバックを反映したもの）
  • 発表資料（スライド）

忘れると怒られます。

指導を受けるときの心得

  • 指示や指摘を受けている時は修正せずに聞くことに徹する。途中で直し始めない
  • 指摘されたら感謝を示す。ボロクソに言われても「ありがとうございます」の姿勢で
  • 提出物は必ず寝かせてから見返す。適当に出すと後で死ぬほど後悔する
  • わからないことは素直に「わかりません」と言う。わかったふりは最悪

先輩からのアドバイス

ボロクソに言われた時のマニュアル: 「とにかく感謝しろ。喜べ。」感謝されて嫌な人はいません。指摘を受け入れる姿勢が大切です。
